Incident #065 - Orders Stuck in Processing

MAJOR ORDER MANAGEMENT

Delayed order execution detected in production pipeline

1. Issue Description

120+ orders stuck in "processing" status for more than 24 hours.

2. Query Investigation

SELECT order_id FROM orders WHERE status = 'processing' AND order_date < NOW() - INTERVAL 1 DAY;

→ 120 delayed orders identified.

3. Payment System Check

SELECT transaction_id, status FROM payment_service WHERE status = 'timeout';

→ Payment API timeout detected.

4. Root Cause

External payment service outage causing queue blockage.

5. Resolution

  • Restarted payment service connection
  • Reprocessed stuck orders
  • Added retry mechanism
